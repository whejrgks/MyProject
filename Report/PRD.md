# 📄 PRD – QR 기반 디지털 메뉴판 시스템

## 1. 제품 개요 (Overview)

### 1.1 제품명

QR 기반 디지털 메뉴판

### 1.2 제품 목적

본 제품은 식당 운영자가 **메뉴를 실시간으로 관리**하고, 고객은 **QR 코드를 통해 편리하게 메뉴를 확인**할 수 있도록 지원하는 디지털 메뉴판 시스템이다.

정적인 종이 메뉴판의 한계를 해결하여 **운영 효율 개선, 고객 경험 향상, 매출 증대 기반 마련**을 목표로 한다.

---

## 2. 문제 정의 (Problem Statement)

* 메뉴/가격/품절 변경 시 즉각 반영 불가
* 고객은 메뉴 정보 부족으로 선택에 어려움
* 직원의 반복 설명으로 운영 효율 저하
* 메뉴 데이터가 축적되지 않아 의사결정 불가

👉 현재 메뉴판은 정보 제공에 그치며, **운영·고객·비즈니스 관점의 문제를 해결하지 못하고 있다**.

---

## 3. 목표 (Goals & Success Metrics)

### 3.1 목표

* 메뉴 관리 시간 50% 이상 감소
* 고객 메뉴 탐색 시간 단축
* 품절로 인한 고객 불만 감소

### 3.2 성공 지표 (KPI)

* 메뉴 수정 → 반영 소요 시간 ≤ 1분
* QR 메뉴판 평균 로딩 시간 ≤ 3초
* 품절 메뉴 주문 발생률 0%

---

## 4. 타겟 사용자 (Target Users)

### 4.1 1차 타겟 – 식당 운영자 / 관리자

* 소형~중형 식당 운영
* 메뉴 변경이 잦음
* IT 전문 지식 없음

**Needs**

* 쉽고 빠른 메뉴 관리
* 실시간 반영
* 안정적인 관리자 페이지

---

### 4.2 2차 타겟 – 식당 방문 고객

* QR 코드 사용 가능
* 메뉴 정보 탐색 니즈 높음

**Needs**

* 보기 쉬운 메뉴
* 충분한 설명과 사진
* 빠른 선택 가능

---

## 5. 사용자 시나리오 (User Journey)

1. 고객이 테이블의 QR 코드를 스캔한다
2. 모바일 브라우저에서 메뉴판이 열린다
3. 메뉴 목록 및 상세 정보를 확인한다
4. 품절 메뉴는 자동으로 제외된다

---

## 6. 기능 요구사항 (Functional Requirements – Gherkin)

### 6.1 고객용 QR 메뉴판

```gherkin
Scenario: 고객이 QR 코드로 메뉴를 조회한다
  Given 고객이 QR 코드를 스캔했을 때
  When 메뉴 페이지가 로드되면
  Then 현재 판매 중인 메뉴 목록이 표시되어야 한다
```

```gherkin
Scenario: 고객이 메뉴 상세 정보를 확인한다
  Given 고객이 메뉴 목록을 보고 있을 때
  When 특정 메뉴를 선택하면
  Then 메뉴 사진, 가격, 설명, 알레르기 정보가 표시되어야 한다
```

```gherkin
Scenario: 품절 메뉴는 노출되지 않는다
  Given 특정 메뉴가 품절 상태일 때
  When 고객이 메뉴판을 조회하면
  Then 해당 메뉴는 비활성화되거나 표시되지 않아야 한다
```

---

### 6.2 관리자 페이지

```gherkin
Scenario: 관리자가 로그인한다
  Given 관리자가 올바른 계정 정보를 입력했을 때
  When 로그인 버튼을 클릭하면
  Then 관리자 대시보드로 이동해야 한다
```

```gherkin
Scenario: 관리자가 메뉴를 등록 또는 수정한다
  Given 관리자가 로그인한 상태에서
  When 메뉴 정보를 저장하면
  Then 변경 사항은 즉시 고객 메뉴판에 반영되어야 한다
```

```gherkin
Scenario: 관리자가 메뉴를 품절 처리한다
  Given 관리자가 메뉴 관리 화면에 있을 때
  When 메뉴를 품절로 변경하면
  Then 고객 메뉴판에서 해당 메뉴는 품절 상태로 표시되어야 한다
```

## 7. 비기능 요구사항 (Non-Functional Requirements)

### 7.1 성능

```gherkin
Scenario: QR 메뉴판은 빠르게 로딩되어야 한다
  Given 고객이 QR 코드를 스캔했을 때
  Then 3초 이내에 첫 화면이 표시되어야 한다
```

### 7.2 가용성

```gherkin
Scenario: 서비스는 영업 시간 동안 중단되지 않는다
  Given 고객이 메뉴판에 접속하면
  Then 서비스는 항상 이용 가능해야 한다
```

### 7.3 사용성

```gherkin
Scenario: 고객은 앱 설치 없이 메뉴를 확인할 수 있다
  When QR 코드를 스캔하면
  Then 브라우저에서 바로 메뉴가 표시되어야 한다
```

### 7.4 보안

```gherkin
Scenario: 관리자 페이지는 인증이 필요하다
  Given 비로그인 사용자가 접근했을 때
  Then 로그인 페이지로 이동해야 한다
```

```gherkin
Scenario: 비밀번호는 안전하게 저장된다
  Then 비밀번호는 해시 처리되어 저장되어야 한다
```

### 7.5 확장성

```gherkin
Scenario: 메뉴 수가 증가해도 시스템은 안정적이다
  Given 메뉴가 500개 이상일 때
  Then 응답 시간은 허용 범위 내에 있어야 한다
```

### 7.6 접근성

```gherkin
Scenario: 고령자도 메뉴를 쉽게 읽을 수 있다
  Then 글자 크기와 대비는 읽기 쉬워야 한다
```
